# Get Started

この練習問題サイトの形式に、今のうちから慣れておきましょう。
いちおう、ほとんどのプログラミングコンテストの形式に則っています。
ディレクトリex0には、基礎的な内容がたくさん含まれています。

## Hello, World!

まずは、伝統のHello Worldからはじめましょう！

- [演習0-001](<./Exercises/ex0/ex0-001.md>)

ページを開いたら、いくつかの項目が見れると思います。
おおまかに、問題と提出欄という感じです。
まずは、問題を見ましょう。問題を理解したら、問題を解くプログラムをエディターに入力して、「提出」ボタンを押してみてください。

「ジャッジ中...」という通知が現れれば、正しく送信できています。それ以外のときは、なんらかのエラーが出た、ということなので、管理者に問い合わせてください。

しばらく待機すると、「AC」、「WA」、「CE」のいずれかの通知が現れます。それぞれの意味は、

- AC ... 正解です！問題を正しく解けています！
- WA ... 不正解です…もう一度解き直しましょう。
- CE ... コンパイルエラーです。ちゃんとプログラムは書けていますか？

みたいな感じです。

Hello Worldなんてもう飽きたよ！という人は、以下のコードをコピペしてみてください。

```c++ linenums="1"
#include <iostream>
using namespace std;

int main()
{
    cout << "Hello, World!" << endl;
    return 0;
}

```

## 複数ファイル

このサイトのユニークな部分として、複数のファイルに対応しています。
そのため、ヘッダファイルをインクルードして…みたいなことができます。
実演してみましょう。

- [演習0-002](<./Exercises/ex0/ex0-002.md>)

さて、この問題を開くと、制約だとか、入力だとか、見慣れないものがあります。
これは、一般的なプログラミングコンテストを意識した問題形式となっています。
そのため、制約など少し仰々しい項目が見られますが、今回はあんまり気にしなくて結構です。

さて、このサイト特有の点として、与えられるファイル、というものがあります。
C++では複数のソースファイルをリンクして実行ファイルを作成することが普通なので、
そのためのファイルが与えられてるよ～ということです。

では、問題を解いていきましょう。ただ、この問題の趣旨は複数のファイルの追加方法を覚える、ということなので、
main.cppの模範解答は事前に提示します。

```c++ linenums="1"
#include "Person.h"
#include <iostream>
#include <string>
using namespace std;

int main()
{
    string s;
    int age;
    cin >> s >> age;
    
    Person p(s, age);
    p.displayInfo();
    
    return 0;
}
```

初学者にとっては分かりづらい部分もありますしね。
これをmain.cppにコピペしましょう。

では本題。ファイルを追加します。
コードのエディターの右上には、「ファイル追加」ボタンがあります。
それをクリックすると、ファイル名の入力を求められるため、まずは「Person.h」と入力してください。

すると、Person.hというファイルが追加されます。
ここに、問題で与えられたPerson.hをコピペしましょう。
問題中の「Person.h」部分をクリックすると、コードが見れます。

Person.cppについても同様にしてください。

ファイル名を間違えちゃったよ～というときは、対応するファイルのバツマークを押せば消せます。
ただし、ファイルが残り1つのときは消去できません。

また、main.cppという名前のファイルは必ず含まれている必要があるので、注意してください。

最後に、提出して、ACが表示されれば正しくファイル追加できています！やったね！
